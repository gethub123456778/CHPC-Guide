# Contributing to CHPC Cluster Documentation

Thank you for your interest in contributing to the CHPC Cluster Usage Guide! This document provides guidelines for contributing to this documentation.

## How to Contribute

### Reporting Issues

If you find an error, have a suggestion, or want to report a problem:

1. **Check existing issues** to see if your problem has already been reported
2. **Create a new issue** with a clear description of the problem
3. **Include relevant information** such as:
   - Error messages
   - Steps to reproduce
   - Your environment (OS, software versions)
   - Expected vs actual behavior

### Suggesting Improvements

We welcome suggestions for:
- New topics to cover
- Improvements to existing content
- Additional examples
- Better explanations
- Code improvements

### Submitting Changes

To submit changes:

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
4. **Test your changes** to ensure they work correctly
5. **Commit your changes** with clear commit messages
6. **Push to your fork**
7. **Submit a pull request**

## Content Guidelines

### Writing Style

- Use clear, concise language
- Write for users with varying levels of experience
- Include practical examples
- Use consistent terminology
- Follow the existing documentation style

### Code Examples

- Ensure all code examples are tested and working
- Use appropriate syntax highlighting
- Include comments for complex code
- Provide complete, runnable examples
- Use realistic but simple examples

### Documentation Structure

- Use clear headings and subheadings
- Organize content logically
- Include table of contents for long documents
- Use consistent formatting
- Cross-reference related topics

## File Organization

### Directory Structure

```
docs/
├── setup/           # Installation and setup guides
├── usage/           # Basic usage guides
├── advanced/        # Advanced topics
├── examples/        # Example scripts and code
├── data/            # Data management guides
├── troubleshooting/ # Troubleshooting guides
└── resources/       # Additional resources
```

### File Naming

- Use descriptive, lowercase names
- Separate words with underscores
- Use `.md` extension for Markdown files
- Use `.sh` extension for shell scripts

## Code of Conduct

### Be Respectful

- Be respectful of all contributors
- Provide constructive feedback
- Help others learn and improve
- Be patient with newcomers

### Be Helpful

- Answer questions when you can
- Share your knowledge and experience
- Provide clear explanations
- Point to relevant documentation

### Be Professional

- Use professional language
- Focus on technical content
- Avoid personal attacks
- Respect different viewpoints

## Review Process

### Pull Request Review

1. **Automated checks** will run on your PR
2. **Reviewers** will examine your changes
3. **Feedback** will be provided if needed
4. **Changes** may be requested
5. **Approval** will be given when ready

### What Reviewers Look For

- Accuracy of information
- Clarity of explanations
- Completeness of examples
- Consistency with existing content
- Proper formatting and structure

## Getting Help

If you have questions about contributing:
